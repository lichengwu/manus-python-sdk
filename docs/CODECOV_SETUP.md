# Codecov Setup Guide

## Quick Start

### Step 1: Sign up for Codecov

1. Visit https://codecov.io/
2. Click "Sign Up"
3. Login with your GitHub account
4. Authorize Codecov to access your repositories

### Step 2: Enable Your Repository

1. In Codecov dashboard, find `lichengwu/manus-python-sdk`
2. Click to enable the repository
3. Go to **Settings** → **General**
4. Copy the **Codecov Token**

### Step 3: Add GitHub Secret

#### Option A: Using GitHub CLI (Recommended)

```bash
# Get your Codecov token first, then run:
gh secret set CODECOV_TOKEN --repo lichengwu/manus-python-sdk
# Paste your token when prompted
```

#### Option B: Using GitHub Web UI

1. Go to: https://github.com/lichengwu/manus-python-sdk/settings/secrets/actions
2. Click "New repository secret"
3. **Name**: `CODECOV_TOKEN`
4. **Value**: Paste your Codecov Token
5. Click "Add secret"

### Step 4: Verify Setup

1. Push a commit to trigger CI
2. Check GitHub Actions: https://github.com/lichengwu/manus-python-sdk/actions
3. Look for "Upload coverage to Codecov" step
4. Visit your Codecov dashboard to see coverage report

## Configuration Files

### `.github/workflows/ci.yml`

```yaml
- name: Upload coverage to Codecov
  uses: codecov/codecov-action@v4
  with:
    files: ./coverage.xml
    flags: unittests
    name: codecov-umbrella
    fail_ci_if_error: false
    token: ${{ secrets.CODECOV_TOKEN }}
    verbose: true
```

### `.codecov.yml`

Coverage settings:
- **Range**: 70% - 100%
- **Precision**: 2 decimal places
- **Status checks**: Project and Patch coverage
- **Ignored files**: tests/, examples/, **/__init__.py

## Troubleshooting

### Coverage not uploading?

1. Check GitHub Actions logs for errors
2. Verify `CODECOV_TOKEN` secret is set correctly
3. Ensure `coverage.xml` is generated by pytest
4. Check Codecov dashboard for repository status

### Token not working?

1. Regenerate token in Codecov Settings
2. Update GitHub Secret with new token
3. Re-run the failed workflow

### Coverage report missing?

1. Check if pytest-cov is installed
2. Verify `--cov=src/manus --cov-report=xml` flags
3. Check `.codecov.yml` for ignore patterns
